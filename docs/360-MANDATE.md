# RobbottX 360-degree mandate

Every owner prompt is the explicit one-percent instruction. Execution must
surface and integrate the missing system without changing the owner's intent.

## Persistent lenses

1. Ownership, strategy, capital, sequencing, and defensibility.
2. Users, jobs, expertise levels, accessibility, psychology, trust, support,
   localization, and education.
3. Robotics science; systems, mechanical, electrical, embedded, software,
   controls, AI, manufacturing, maintenance, and simulation engineering.
4. Identity, ontology, evidence, units, licensing, provenance, freshness,
   deduplication, revisions, graph relationships, and compatibility rules.
5. Information architecture, original visual design, responsive interaction,
   3D, static fallbacks, WordPress, performance, and observability.
6. SEO, entity authority, internal linking, anti-cannibalization, structured
   data, sitemaps, facets, AI discovery, feeds, and multilingual search.
7. Procurement, offers, inventory, suppliers, landed cost, RFQs, marketplace,
   payments, fulfillment, warranties, and lifecycle services.
8. Safety, ethics, law, privacy, security, export controls, dangerous use,
   counterfeit defense, certification boundaries, and incident handling.
9. Planetary environments, autonomy, communication delay, repairability,
   reuse, planetary protection, local manufacturing, and ISRU.
10. Adversarial review, metrics, QA, change control, and reusable learning.

## Working cycle

`literal request -> expansion -> current research -> system impact -> smallest
sound slice -> implementation -> evidence -> learning`

360-degree thinking does not mean uncontrolled simultaneous construction. It
means every increment is designed so the complete enterprise can extend it
without rebuilding the foundation.
