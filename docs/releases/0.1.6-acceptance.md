# RobbottX Core 0.1.6 acceptance contract

## Purpose

Version 0.1.6 closes the remaining public exposure of the exact inherited
WordPress routes and the empty WooCommerce route set. Those records remain
available to administrators and editors, but unauthenticated visitors receive
HTTP 410 through the established theme error surface.

The release deletes no records, removes no commercial offer because no reviewed
offer exists, and makes no new technical or availability claim. The shop, cart,
checkout, and account routes remain inactive until a later release can prove a
real product catalog and exact supplier, region, quantity-basis, and freshness
evidence for every purchase control.

The paired presentation package is RobbottX theme 0.1.5. It adds an in-body
theme identity and a stable atlas-home class so the theme transition can be
proven from the rendered body without changing the visible layout.

## Required behavior

- Register the route transition on `template_redirect` before canonical
  redirects.
- Apply it to the inherited robot type, exact default pages and post,
  Uncategorized archive, retired robot archive, and every public author
  discovery surface under the deliberate site-wide author-privacy policy.
- Apply it to the empty shop, cart, checkout, and account pages, including
  HTML, page sitemap, REST collection, REST detail, automatic page list, and
  navigation discovery.
- Treat the WooCommerce product archive as the shop route while preserving
  product records in administration for a later active-commerce release.
- Keep product details, product collections, product taxonomies, Store API
  product discovery, search, feeds, sitemaps, and navigation unavailable to
  unauthenticated visitors in the inactive-commerce state.
- Preserve administration and users with editorial permission.
- Mark the main query as 404 so the active theme renders its error template.
- Return HTTP 410 and disable response caching.
- Disable WordPress canonical redirects after the retired request is marked.
- Disable the separate WordPress old-slug redirect path after the retired
  request is marked.
- Preserve the internal retired-request flag after the query transition so
  WordPress and Rank Math output noindex without conflicting directives.
- Fail closed with a generic 410 response if the expected WordPress query
  object is unavailable.

## Source acceptance evidence

- Every changed PHP file passes `php -l`.
- The public-discovery harness proves all retired request types return 410,
  use the error template, disable caching, and remain noindex.
- The harness proves product records, administration, and editorial contexts
  remain available to authorized users while every inactive commerce surface
  stays private.
- Full repository QA passes.
- Two independent package builds produce the same ZIP SHA-256.
- ZIP inspection proves version 0.1.6, the route-transition code, the stable
  plugin root, and the absence of secrets.

## Live acceptance evidence

- The health endpoint and rendered body report plugin version 0.1.6.
- The active theme API reports version 0.1.5, the rendered body reports the
  matching theme identity, and the previous homepage markup is absent.
- Every exact inherited HTML route returns 410 with established English
  error content and no inherited title or link.
- REST collections, REST details, REST search, HTML search, feeds, robots.txt,
  and every advertised sitemap pass the immutable live verifier.
- Shop, cart, checkout, and account return exact uncached noindex 410 responses.
- Homepage, desktop, and mobile acceptance gates pass.
- The literal hosting-root `robots.txt` matches `hosting/robots.txt`, returns
  HTTP 200 as `text/plain`, permits the public site, and advertises the verified
  WordPress sitemap.
- The temporary deploy route is deleted and the temporary Application Password
  is revoked immediately after verification.
