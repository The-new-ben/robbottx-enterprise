# Precision Atlas design direction

RobbottX should feel like a living engineering atlas: museum-grade visual
quality, laboratory-grade evidence, and procurement-grade usefulness.

## Original visual language

- Mineral white, fog, and graphite form the neutral field.
- Cool cyan/mint means verified relationships.
- Amber is reserved for uncertainty.
- Red is reserved for contradiction or incompatibility.
- Identifiers, protocols, versions, and units use a restrained monospace
  treatment; normal reading stays in a humanist system stack.
- The signature component is an evidence ribbon paired with graph rails that
  connect mission, system, module, component, software, source, and offer.
- Motion explains relationships and state changes. It must have keyboard and
  reduced-motion equivalents.

## Homepage sequence

1. Clear identity and universal search.
2. One precise promise: a verified map of robotics systems.
3. Evidence ribbon using only real counts.
4. System map from mission through offer, with edge state.
5. Explore, compare, configure, and source paths.
6. The first mobile-manipulator golden slice.
7. Methodology, corrections, source-language coverage, and change history.
8. Commerce only after canonical identity and evidence.

## Entity page sequence

1. Canonical identity, native labels, manufacturer, family/model/revision,
   lifecycle, RobbottX ID, evidence grade, and verification date.
2. Media plus a real family/revision selector.
3. Normalized quick facts with unknowns shown explicitly.
4. Overview, specifications, relationships, compatibility, evidence,
   documents/media, offers, and change log.
5. Semantic BOM/graph fallback when 3D is unavailable.
6. Per-dimension compatibility with source, condition, ruleset, and time.

## Anti-copy rule

Benchmark lessons are abstract principles. Each adopted lesson must be
transformed in at least three dimensions among information architecture,
visual form, palette/type, interaction, copy, and imagery. Do not copy
slogans, section sequences, palettes, proprietary assets, card geometry,
iconography, or animation timing.
