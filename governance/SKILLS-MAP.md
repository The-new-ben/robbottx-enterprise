# Applied skills map

## Transferred RobbottX operating skills

- `expand-enterprise-prompts`: expands the owner's literal one-percent request
  through the persistent 360-degree lenses before execution.
- `build-robbottx`: enforces evidence, hierarchy, graph, compatibility,
  WordPress boundaries, multilingual readiness, SEO discipline, QA, receipts,
  and truthful completion.
- `robbottx-public-language`: defines the public audience, established-platform
  voice, progressive disclosure, technical state vocabulary, claim boundaries,
  and release rejection rules for all customer-facing language.
- `robbottx-proprietary-model-360`: controls original robot identity, measurable
  requirements, benchmark boundaries, evidence classes, real-part selection,
  interfaces, BOM lineage, safety and security gates, 3D truth, verification,
  IP review, and the private-to-public boundary.
- `robbottx-commerce-evidence-release`: keeps empty commerce private and permits
  a public catalog, product, availability statement, or purchase control only
  when exact product and current offer evidence pass the buyer, supplier,
  region, inventory, SEO, accessibility, and live-release gates.

## WordPress execution skills

- `wordpress-router`: routes work to the minimum required WordPress specialist
  instructions.
- `wp-block-themes`: controls the theme structure, `theme.json`, templates,
  parts, patterns, and fallback behavior.
- `wp-plugin-development`: controls plugin structure, lifecycle, validation,
  data boundaries, and additive migrations.
- `wordpress-agent-deploy`: controls deterministic packages, exact public
  artifacts, temporary capability-gated deployment routes, independent
  verification, cache handling, cleanup, and emergency recovery.
- `robbottx-live-release-acceptance`: separates code installation from public
  acceptance and proves release identity, route cleanup, inherited-content
  suppression, commerce, SEO, language, performance, accessibility, and
  exact-viewport evidence.
- `wp-performance`: controls repeatable baselines, warmed samples, fixed test
  scenarios, and median-based performance evidence.

## Browser and visual skills

- `chrome:control-chrome`: authenticated production inspection and later
  release execution.
- `screenshot`: exact-viewport evidence and visual comparison.

The owner explicitly overrides the generic staging and backup requirements.
That override is recorded in `docs/DIRECT-PRODUCTION-POLICY.md`; it does not
remove local QA, evidence, or reversibility requirements.
